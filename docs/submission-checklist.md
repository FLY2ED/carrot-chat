# Submission Checklist

당근 채팅팀 인턴 지원 제출 전 체크리스트입니다.

## 필수 링크

- [ ] `carrot-chat` 라이브 URL 접속 확인: https://carrot.naldadev.com
- [ ] `carrot-chat` GitHub 저장소 공개 URL 입력
- [ ] GitHub README 첫 화면에 프로젝트 목적, 기술 스택, 테스트 명령이 보이는지 확인
- [ ] 공개해도 되는 보조 프로젝트 URL만 이력서에 포함
- [ ] 민감한 계정/토큰/환경변수/클라이언트 정보가 README와 docs에 없는지 확인

## 제출 문장

- [ ] `docs/application-draft.md`에서 학교/전공/근무 가능 여부 반영
- [ ] 지원동기에 "당근 채팅은 여러 서비스가 만나는 접점"이라는 이해가 들어갔는지 확인
- [ ] 프로젝트 설명에 SDK 경계, WebSocket, 테스트 전략, 서버측 정책 제어가 모두 들어갔는지 확인
- [ ] 과장 표현 제거: "당근 내부 SDK와 동일" 같은 표현 금지
- [ ] 정량 수치는 실제 확인 가능한 것만 입력

## 이력서 프로젝트 섹션

- [ ] 대표 프로젝트는 `carrot-chat`을 첫 번째로 배치
- [ ] 역할을 "기획/설계/구현/테스트/배포" 중 실제 수행한 범위로 명시
- [ ] 검증 결과를 짧게 표기: typecheck, unit/component 14 tests, E2E 2 tests
- [ ] 보조 프로젝트는 2~3개만 선택해 과밀하게 보이지 않게 정리
- [ ] `branches` 폴더 작업은 핵심 근거에서 제외

## 기술 질문 대비

- [ ] 왜 UI와 SDK를 분리했는지 설명 가능
- [ ] 왜 Durable Objects를 썼는지 설명 가능
- [ ] 재연결 로직에서 의도적 종료와 비정상 종료를 왜 구분했는지 설명 가능
- [ ] 서버측 연락처 마스킹 정책의 한계와 확장 방향 설명 가능
- [ ] SDK로 배포하려면 추가로 무엇이 필요한지 설명 가능

## 마지막으로 사용자에게 필요한 정보

- [ ] 2026-06-29부터 3개월 풀타임 근무 가능 여부
- [ ] 학교/전공/학년 또는 졸업 상태
- [ ] 공개 GitHub URL
- [ ] 대표 프로젝트별 정확한 기간과 팀 규모
- [ ] 실제 성과 수치
- [ ] 이력서 PDF 최신본
